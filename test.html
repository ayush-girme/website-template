Core Component Architecture

System Matrix
Phase 01

Intent & Translation

Action: Ingests unstructured human commands and forces the AI model to output strictly formatted JSON payloads.

Reason: Translates ambiguous goals into deterministic, executable system arguments before any code is written.

Phase 02

Sandbox Execution

Action: The local runtime wrapper parses the JSON and executes generated scripts inside an isolated background terminal.

Reason: Security and stability. Untested code is never executed on the main thread; the subprocess acts as a secure firewall.

Phase 03

Self-Correction Loop

Action: Automatically captures 'stdout' and 'stderr'. If 'stderr' triggers, the agent reads its logs and refactors the code autonomously.

Reason: Achieves true zero-human-lag automation by handling its own syntax errors without manual debugging.

Phase 04

State Commits

Action: Successful executions are logged into 'session_tracker.md' and securely pushed via Git.

Reason: Leverages local file systems and Git history to create a flawless, secure audit trail without expensive vector databases.
